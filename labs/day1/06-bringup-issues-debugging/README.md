# Lab 1-6 — Bring-up Issue Debugging

## Objective

Practice diagnosing typical early board issues.

## Scenario examples

- No UART output
- Bootloader works, kernel has no console
- Kernel starts but rootfs cannot mount
- Device driver never probes
- Driver probes but hardware does not respond
- Deferred probe repeats for many seconds

## Tasks

1. Pick one scenario.
2. Write the likely root-cause tree.
3. Identify the first three commands or checks.
4. Decide what evidence would prove or disprove each hypothesis.

## Deliverable

A short debugging playbook.
